# 提示词处理器说明

## 工作原理

这个提示词处理器实现了半自动化的信息补全系统，类似 Cursor 的交互方式，但更加智能化。

### 处理流程

```
1. 用户提出问题
   ↓
2. AI 分析意图和所需信息
   ↓
3. 自动提取已知信息（项目结构、技术栈等）
   ↓
4. 询问缺失的关键信息
   ↓
5. 生成完整的背景任务
   ↓
6. 执行任务并给出结果
```

## 具体实现示例

### 场景1：开发任务

**用户输入**：
```
"添加一个用户评论系统"
```

**AI 处理过程**：

1. **意图分析**：
   - 类型：开发任务
   - 需要：新功能实现
   - 相关：前端、数据存储

2. **自动提取信息**：
   ```javascript
   {
     project: "VuePress 2.x + Vercel",
     hasApi: true,
     scripts: ["publish", "push"],
     existingFeatures: ["文章发布", "封面生成"]
   }
   ```

3. **智能提问**：
   ```
   "我来帮你添加评论系统。根据你的项目情况，我了解到：
   - 技术栈：VuePress 2.x + Vercel
   - 已有 API 功能
   - 已有文章系统

   需要确认几个问题：
   1. 评论数据需要持久化存储吗？（例如存到数据库或文件）
   2. 希望支持哪些评论功能？（发表、回复、删除、审核）
   3. 需要用户登录才能评论吗？
   
   请选择或提供详细信息。"
   ```

4. **背景任务生成**：
   ```markdown
   ## Background Context
   - **Project**: VuePress 2.x + Vercel 静态网站
   - **Task**: 实现用户评论系统
   - **User**: 开发者
   - **Environment**: Vercel 部署
   - **Constraints**: 需要考虑静态网站的存储限制
   - **Requirements**: 持久化、基本CRUD功能
   ```

### 场景2：调试任务

**用户输入**：
```
"文章封面生成失败"
```

**AI 处理过程**：

1. **意图分析**：
   - 类型：问题调试
   - 相关：AI封面生成功能

2. **自动提取信息**：
   ```javascript
   {
     project: "VuePress 2.x",
     hasCoverGeneration: true,
     apiRoutes: ["publish", "cover"],
     lastDeployment: "2026-04-21"
   }
   ```

3. **智能提问**：
   ```
   "我来帮你解决封面生成问题。我了解到你的项目有AI封面生成功能。

   请提供以下信息：
   1. 具体的错误信息是什么？
   2. 是所有文章都失败还是特定文章？
   3. 最近有修改过配置或代码吗？
   4. 在本地还是线上环境出现的问题？
   
   这些信息会帮助我快速定位问题。"
   ```

## 信息收集策略

### 🔍 自动收集（无需用户输入）
- **项目信息**：从 package.json、配置文件获取
- **技术栈**：从文件结构判断
- **环境信息**：从运行时获取
- **历史信息**：从对话记录获取

### 💬 智能询问（按需提问）
- **任务相关**：根据任务类型询问必要信息
- **上下文敏感**：基于已有信息决定还需要什么
- **优先级排序**：先问最重要的信息

### 🤝 用户体验优化
- **一次只问一个问题**
- **提供默认选项**
- **解释为什么需要这个信息**
- **允许跳过非关键问题**

## 背景任务生成

### 必填字段
```yaml
Background:
  Project: 项目名称和类型
  Context: 当前开发状态
  User: 用户角色
  Task: 任务描述
  Constraints: 已知限制
```

### 可选字段
```yaml
Background:
  Timeline: 时间要求
  Priority: 优先级
  Dependencies: 依赖关系
  Resources: 可用资源
```

## 与 Cursor 的区别

### 相似点
- 都通过问答式交互
- 都会主动补全信息
- 都关注上下文理解

### 不同点
- **更智能化**：自动收集更多信息
- **更灵活**：根据任务类型调整策略
- **更全面**：考虑整个项目环境
- **更协作**：保持对话自然流畅

## 使用建议

### 对于用户
1. **简单描述**：直接说你要做什么
2. **逐步回答**：按我的问题回答
3. **随时补充**：觉得信息不足可以主动提供

### 对于AI助手
1. **快速理解**：第一时间理解用户意图
2. **主动收集**：自动获取项目相关信息
3. **精准提问**：只问必要的信息
4. **自然交互**：保持对话的流畅性

## 实际应用

### 开发任务
```
用户："添加搜索功能"
↓
AI："添加搜索功能。你的项目是VuePress网站，有API支持。

需要确认：
1. 搜索范围（文章/标签/全站）
2. 实时搜索还是搜索后提交
3. 性能要求

请选择。"
```

### 修复任务
```
用户："登录页面打不开"
↓
AI："登录页面问题。我了解到你有VuePress项目。

请提供：
1. 错误提示信息
2. 页面完全加载了吗？
3. 什么时候开始出现的？

这些信息帮助我快速定位问题。"
```

### 优化任务
```
用户："网站加载很慢"
↓
AI："网站性能优化。项目是VuePress + Vercel。

自动信息：
- 已有图片优化
- 有CDN支持
- VuePress构建已启用

需要确认：
1. 主要慢在哪个页面？
2. 大概的加载时间？
3. 用户反馈如何？

请提供详细信息。"
```
