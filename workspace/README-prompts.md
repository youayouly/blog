# 提示词系统说明

## 概述
这个目录包含了半自动化的提示词处理系统，用于 AI 助手与用户进行高效的交互式对话。

## 文件结构

```
workspace/templates/prompts/
├── interaction-template.md    # 交互式提示词模板
├── prompt-processor.md       # 提示词处理器说明
├── auto-collector.js         # 自动信息收集器
├── prompt-handler.js         # 提示词处理器
└── README-prompts.md         # 本文件
```

## 工作原理

### 自动信息收集
系统会自动收集以下信息：
- 项目技术栈（VuePress、框架等）
- 可用脚本和命令
- 文件结构
- 环境信息
- 约束条件

### 智能提问
根据用户意图，智能生成相关问题：
- **开发任务**：功能范围、技术选择、特殊要求
- **调试任务**：错误信息、复现步骤、环境
- **优化任务**：优化目标、性能指标、期望效果
- **部署任务**：部署环境、配置、域名

### 背景任务生成
基于收集的信息，生成完整的任务背景：
```yaml
Background:
  Project: 项目信息
  Context: 当前状态
  User: 用户角色
  Task: 任务描述
  Constraints: 已知限制
```

## 使用方式

### AI 助手使用
1. 用户提出问题
2. 系统自动收集项目信息
3. 分析用户意图
4. 智能提问补全信息
5. 生成背景任务
6. 执行任务并给出结果

### 用户使用
1. **简单描述**：直接说你想做什么
   - 例如："添加搜索功能"
   - 例如："登录页面报错"

2. **回答问题**：按 AI 的提示回答
   - 只回答必要的信息
   - 可以提供额外细节
   - 不确定的选择默认选项

3. **获得结果**：AI 会给出具体的解决方案

## 扩展性

### 添加新的任务类型
1. 在 `prompt-handler.js` 中添加新的分析逻辑
2. 对应添加问题生成方法
3. 更新响应模板

### 添加新的项目类型
1. 修改 `auto-collector.js`
2. 添加新的技术栈检测
3. 更新约束条件

### 自定义交互流程
1. 修改问题生成策略
2. 调整响应模板
3. 添加新的交互模式

## 注意事项

1. **项目信息准确性**：确保项目文件存在且格式正确
2. **环境感知**：正确识别运行环境和可用工具
3. **上下文保持**：维护对话历史和用户回答
4. **性能考虑**：避免过度收集不必要的文件信息
